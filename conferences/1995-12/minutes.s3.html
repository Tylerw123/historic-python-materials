The Python SSLeay interface (Andrew Kuchling)

  • Problem: Credit card numbers in clear are bad
  • lot's of proposed solutions proposed today
  • Current leader is SSL by netscape
  • client/server negotiate an algorthm and session key
  • optionally, clients and servers can authenticate each other
  • provides easy security
  • SSL relys on the X.509 framework - certification authorities Tree based (unlike PGP)
  • SSLeay high level algorithm
    • Server only: generate and RSA key and keep it
    • Call SSLeay.fromfd() to create and SSLeay object attached to socket
    • Server only: set a public key and certificate with appropriate functions
    • call connect or accept
  • SSLeay support was added to Grail in two hours (easy to use!)
  • SSL makes it easy to gain security
  • Python makes it easy to use SSLeay because of Polymorphism
  • SSLeay is still in alpha state
  • interfaces to other SSL implementation could be written
  • because of RSA's RSAref toolkit, SSLeay can be used without patten infringment. Currently not a problem (especially abroad since patten law only applies in the US)

SYLU 1.0 - ILU runtime reimplemented in/for Python (Scott Hassan)

  • What is ILU
    • Invented at Xerox PARC
    • Distributed Objects
    • ISL (Interface Specification Language) - language independent object specifier
    • Every parameter is strongly typed
    • Corba ORB (Object Request Broker)
    • language independent
    • transport independent (UDP, TCP, email)
  • SYLU
    • Scott's Yet another Language Unification
    • Just the Python/ILU runtime reimlemented
    • Under 4000 lines of code
    • doesn't implement entire ILU 1.X - no courier protocol, no singleton types, no garbage collection, no UDP
    • Future
      • ILU 2.0 support
      • thread support for big time server programs
      • support encryption that ILU supports (SSL, others)
    • Grail applications that use SYLU
      • Chat room
      • Drawing room - everyone draws on the same page
      • Four player Roulette game
      • http://www-db.standford.edu/~hassan/grailApps
    • Performance - ILU will be faster than raw HTTP because the connection stays open (adds 25% overhead to raw TCP)
